I think it would be much stronger if it started with the real problem that motivated this project.

This project was not created because I wanted another cleanup script.

It was created because I encountered a real debugging problem.

After several weeks of intensive Codex usage, I noticed that my Mac's "System Data" had grown unexpectedly.

Like many developers, my first assumption was that Xcode or iOS Simulator was responsible.

I spent time investigating the usual suspects before eventually discovering that a significant amount of storage was actually coming from Codex runtime artifacts, especially under ~/.codex.

At that point I realized the real problem wasn't deleting files—it was understanding what Codex had generated, why those artifacts existed, and whether they were safe to remove.

That realization became the motivation for Codex Doctor.
