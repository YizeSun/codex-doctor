Please create a new Codex skill named codex-doctor.

Goal:
Build a self-diagnostic and cleanup skill for Codex CLI environments.

Requirements:
1. The skill must inspect ~/.codex, especially:
   - sessions size
   - largest rollout-*.jsonl files
   - logs_2.sqlite, logs_2.sqlite-wal, logs_2.sqlite-shm sizes
   - ~/Library/Caches/com.openai.codex
2. It should also inspect common developer caches:
   - ~/Library/Developer/Xcode/DerivedData
   - ~/Library/Developer/CoreSimulator
   - pip cache
   - npm/pnpm/yarn cache
   - Homebrew cache
3. Default mode must be read-only diagnosis.
4. Cleanup must require explicit flags, for example:
   - --prune-codex-sessions-days 30
   - --clear-codex-cache
   - --clear-xcode-derived-data
   - --erase-simulators
5. Never delete source code, project folders, Documents, Desktop, or Downloads.
6. Before destructive actions, print an exact cleanup plan and require confirmation unless --yes is passed.
7. Generate a report with:
   - total reclaimable space estimate
   - safe cleanup items
   - risky cleanup items
   - recommended next commands
8. Add SKILL.md explaining when Codex should use this skill.
9. Add shell scripts with careful comments and safe defaults.
10. Add a README with examples:
   - diagnose only
   - prune sessions older than 30 days
   - clear Codex cache
   - clean Xcode DerivedData
